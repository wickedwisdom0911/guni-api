This documentation covers endpoints for MMS, SMS, Campaigns, Scheduling, and Webhooks.

MMS

Endpoints for sending MMS messages.

Send Bulk MMS

Sends bulk MMS messages. Data can be provided via query parameters or as multipart form-data.

Authorizations:
BearerAuth
query Parameters
media
string

Media URL or file identifier.

message
string

MMS message text.

sender
string

Sender ID.

contacts
string

JSON array (as string) of contact numbers.

replyStopToOptout
string

Flag to opt out when a "stop" reply is received.

Request Body schema: multipart/form-data

Multipart form-data payload.

media
string <binary>
message
string
sender
string
contacts
string
replyStopToOptout
string

Responses

Main API Server

https://apit.gunisms.com.au/api/v1/gatewaymms/bulk

Vonage Server

https://apitgwvonage.gunisms.com.au/gatewaymms/bulk

Request samples

  • Payload
Content type
multipart/form-data
{
  "media": "(file)",
  "message": "please ignore test bulk mms shivam",
  "sender": "#SharedNum#",
  "contacts": "[\"405650337\"]",
  "replyStopToOptout": "true"
}

Campaign Schedule

Endpoints for campaign SMS and scheduling.

Campaign SMS / Scheduled Campaign

This endpoint is used to send an immediate campaign SMS or schedule a campaign. When scheduling, include additional scheduling fields in the payload and add the query parameter mode=Mobile.

Authorizations:
BearerAuth
query Parameters
mode
string
Value: "Mobile"

Set to "Mobile" when scheduling a campaign.

Request Body schema: application/json

JSON payload for a campaign SMS or a scheduled campaign.

One of
name
required
string
sender
required
string
message
required
string
contacts
required
Array of strings

Responses

Cancel Recurrence Schedule

Cancels a scheduled campaign. Provide scduleid to cancel the entire schedule. Optionally, include queueid to cancel a single occurrence.

Request Body schema: application/json

JSON payload for cancellation.

string
